# services/library — Stage-2 reading + feed review

Once papers are in your library, this domain ranks what to read next and powers
the deeper review/annotation surfaces, plus the Phase 1.14 feed-review queue.

```
unread library items ─reading_queue→ ranked "Read next" (gate score + reason)
        click ─> deep_review ─┬─ quality_review  (referee-grade digest, 11-part)
                              ├─ quality_eval     (reference-free 3-band FLAG/NEUTRAL/HIGHLIGHT)
                              └─ _paper_goal_summaries (per-goal hybrid retrieval → grounded summary)
        review_fleet ─> pre-decide a ProposedVerdict for the top-N (Confirm/Override, no LLM)
        paper_render ─> _paper_read_brief: at-a-glance Goal Match Board + Quality Panel (iframe)
        border_cache: cached active-learning "border" picks for Annotate
   feeds awaiting review ─review→ approve / reject / relabel ─> golden + pending
   review_detail: uniform "why this score?" payload for /api/golden/review-detail
```

**Goal-aligned brief (deep_review enrichment):** for each requested paper `deep_review`
runs three independently-skippable layers over the extracted text — the digest
(`quality_review`), a reference-free **quality verdict** (`quality_eval`), and **per-goal
grounded summaries** (`_paper_goal_summaries`) — and caches `{digest, quality, goal_summaries}`
in `deep_reviews.json`. The render reads them back and `_paper_read_brief` paints, in
**decision order**, the read verdict (the flag reason inlined for a flagged paper) → the
ARR soundness-vs-relevance spine (Rigor + Relevance chips) → a 6-cell goal board that is the
**single home of per-goal relevance** (HIT/MISS/NOT-RETRIEVED; fired cells carry the grounded
summary + key sections + an on-demand quote) → a **self-explaining Quality panel** (plain-language
band gloss + a one-clause "how we judged it" method line + an N-of-M self-consistency count + a
legend + the decisive signals, red flags first; the full rubric sits behind one disclosure). The
brief is a triage aid, **not** a copy of the paper — the full body is not embedded; the digest is
collapsed by default. A failed layer degrades to no panel, never blocking the digest.

| `deep_review.py` | on-demand full-text deep review of top picks; per item runs `quality_review` (digest) + `quality_eval` (quality verdict) + `_paper_goal_summaries` (per-goal board) as **independently-skippable** layers (a failed layer logs + degrades, never blocks the digest), caching `{digest, quality, goal_summaries}`. The "endpoint … may be unreachable" hint is appended **only** when the per-item error actually looks like a connectivity failure (`_looks_like_connectivity`); a parse/validation error — where the endpoint clearly responded with a bad body — is reported verbatim rather than mislabelled a network outage. A digest that comes back as a raw string instead of a `PaperDigest` (onprem's parse-failure fallback) is salvaged with `extract_json_blob` in `quality_review.assess_digest`, raising cleanly on a truly empty completion rather than crashing on `.model_copy()`. `start()` returns an `accepted` flag (did THIS call claim the single-flight slot) so the review-fleet can tell its own job from a foreign one holding the latch. Cost is **tier-aware** (keyed on the resolved provider's `lean_deep_review` flag — NOT `is_local`, since MLX is loopback yet fast): a provider flagged `lean_deep_review` (ollama, prefill-bound) uses `quality_review.lean_max_text_chars` + `lean_self_consistency_runs` + batched goal summaries (~4 LLM calls instead of ~11, ~1.5–2 min), while every other provider — incl. MLX — uses the full `max_text_chars` + `self_consistency_runs` + per-goal summaries. The lean→full switch (text cap, rubric runs, AND goal batching) is gated identically in the background job and the `verify-deep-review` CLI. Orthogonally, **sub-call concurrency** (the N self-consistency rubric samples + the per-goal summary calls) runs them **in parallel on a remote provider** — capped by the provider's `max_sub_concurrency` (default `TRIAGE_JOB_CONCURRENCY`) — and **serial on a local one** (RAM safety), via the shared `_common.deep_review_sub_concurrency` helper used by BOTH the job and the CLI so they can't drift. On a slow remote *reasoning* model (e.g. kather `sota`, ~30–75 s/call of thinking) this is the dominant win: ~11 serial calls collapse toward `extract + digest + max(rubric) + overstate + max(goals)`, with **identical prompts/aggregation** (speed only, no quality change). **Per-phase thinking:** the DIGEST runs with reasoning ON (`resolve_stage_client("deep_review", enable_thinking=True)` → `llm_digest`), while rubric/overstate/goals keep the provider's fast thinking-OFF default — a benchmark vs the pinned 397B judge (`tools/bench_deep_review.py`) showed a thinking-OFF digest goes near-empty or hallucinates numbers, while the trivial calls don't need reasoning (overstate 165 s→4 s). On kather `sota` this lands a faithful, complete `--with-goals` review in ~60 s. `_review_one` reviews from `item['pdf_path']` when set (the review-fleet's `pdf_overrides` — an acquired cache download) ELSE the Zotero attachment `detail['pdf_path']`, so a verdict needs no Zotero write. The three enrichment layers live in `_deep_review_layers.py` (split out to stay under the 500-LOC cap) |

| `review_fleet/` | **Phase-2 background fleet** that PRE-DECIDES a reading verdict for the top-N Read-next picks so the human only Confirms/Overrides. | `quality_review.py` | full-text, **referee-grade** quality digest (NeurIPS/ICLR rubric: contribution, soundness-with-rationale, key evidence, novelty vs prior work, strengths, weaknesses/limitations, reproducibility + 1–5 scores + a one-line `verdict`) PLUS the reader's 11-part `summary_structure` (executive summary, key findings, methods, limitations, industry/academy impact), grounded in the paper's own text. Editable via `goals.yaml: prompts.paper_digest` (default `_DEFAULT_DIGEST_PROMPT`). The full text is selected by `_review_text.select_review_text` (budget-aware) instead of a blind `[:cap]` prefix slice, so over-cap papers keep their referee-critical tail instead of dropping it. `_DEFAULT_DIGEST_PROMPT` opens with a hard **ANTI-FABRICATION** rule (only numbers/URLs/entities VERBATIM in the paper; an empty field beats an invented one) — the decisive fix for weak LOCAL models, which otherwise fabricate statistics to fill `key_findings` (benchmarked: qwen3:8b faithfulness 1→9 on a perspective paper) |
| `paper_type.py` / `_paper_type_checklists.py` | **paper-TYPE detection + per-type, standard-grounded CHECKLISTS** (the fix for "a review gets flagged for no-ablation/no-leakage"). `paper_type.detect` is layered + lazy: cheap structural signals + a Zotero itemType prior → ONE LLM classification call → low-confidence/contradiction falls back to a safe SUPERTYPE (`generic_empirical`/`generic_review`), never guesses; a per-item user override wins. `_paper_type_checklists` holds the 14-type MECE taxonomy + a `CHECKLISTS[type]` table of binary YES/NO/NA items, **each citing its standard + a `source_url`** (TRIPOD+AI/PROBAST/CLAIM/STARD-AI/CONSORT-AI clinical · NeurIPS/REFORMS/Kapoor-Narayanan-leakage empirical · PRISMA/AMSTAR-2 systematic review · SANRA narrative/survey · Wachsmuth argument-quality position/policy · CARE case report), plus the **pure `coverage_grade`**: weighted % of APPLICABLE items met (N/A excluded, critical items double-weighted, a failed critical item or red flag caps the band) — the transparent replacement for the unvalidated 1-5 LLM self-scores. `test_paper_type_checklists.py` asserts every item carries a real source URL. |
| `quality_eval.py` / `_quality_prompts.py` | **reference-free, author-blind, TYPE-AWARE** full-text quality verdict — a coarse 3-band `flag`/`neutral`/`highlight` (never a fine score). `evaluate_quality(..., paper_type=…)` selects the `CHECKLISTS[type]` rubric (a review is judged on SANRA/PRISMA, never ablations), runs it 3× (self-consistency, bias-guarded, each `yes` needs a grounded quote — **fuzzy/paraphrase-tolerant** via `_grounding.quote_is_grounded(fuzzy=True)`, since the rubric LLM rarely quotes verbatim; the strict bar collapsed coverage to ~0), and derives the band + A–D `grade` from `coverage_grade` (coverage fraction, NOT an LLM number); new `coverage_*`/`missing_critical`/`paper_type`/`coverage_standard` fields say what standard was applied. The structural **leakage red-flags fire ONLY for EMP-family types** — a review citing a high number no longer trips them; and the near-perfect-metric red-flag (`_near_perfect_metric`) only counts a near-1 value (0.98/0.999/99%/100%) as a HEADLINE METRIC when a performance-metric word sits within ~40 chars of it, so an optimizer hyperparameter (Adam β2 = 0.98/0.999, present in every modern ML paper) is no longer mistaken for one and can't cap a methods paper like Transformer/BERT to `flag`. Near-duplicate red flags merge (`_dedupe_near`, token-Jaccard). RIGOURATE abstract-vs-body overstatement check still runs; `uncertain` when self-consistency disagrees. Prestige no longer affects the band (it is a ranking signal). **Self-verification 2nd pass** (`_self_verify`, `quality_review.self_verification`): one short LLM call re-checks the CRITICAL items marked met — does the grounding quote actually establish the criterion? — and DEMOTES over-claims (the LLM positivity bias) to missing-critical, recorded in `self_verification_demoted`. The prompt is **confirm-by-default** (reject only when CONFIDENT the quote fails the criterion) so it stops demoting correct grounded criticals — the earlier "skeptical reviewer" framing collapsed even gold-quality papers to `flag` (caught by the benchmark). Likewise the abstract-vs-body **overstatement** red-flag now needs a cluster (≥3, was ≥2) of CLEAR over-claims to cap the band, so 1-2 false-positive overstatements can't sink a fully-covered paper. **Phase A encoder shadow:** when `quality_review.shadow_claim_check` is on, `_shadow_claim_scores` also runs the MiniCheck ENCODER (`services/model/claim_checker`) over the SAME retrieved evidence the LLM saw and records per-claim support probs in `QualityEval.claim_support_probs` — a reproducible encoder-vs-LLM A/B, NO behavior change (band/overstatements stay LLM-decided). `claim_grounding_rate` is still reserved for the faithbench aspect track (follow-up). The LLM-fed rubric `body` is chosen by `_review_text.select_review_text` (budget-aware, not a blind prefix); the structural/domain/red-flag checks still scan the FULL text. The N self-consistency samples are dispatched **concurrently** when `sub_concurrency > 1` (remote tier) — identical prompts, order-deterministic aggregation — and serially when 1 (local) |
| `_paper_goal_summaries.py` | **goal-conditioned** per-goal summaries for the board. Section-aware chunking (reuses `faithbench.chunk_text`/`tokenize`) → per-goal hybrid retrieval (a **stronger** lazy dense embedder `BAAI/bge-large-en-v1.5` + BM25 → `_search._rrf` → `reranker.rerank`, degradation ladder) → dense-cosine relevance **gate** → facet abstraction (local LLM) of fired goals only → per-quote grounding (`_grounding`). Returns one `GoalSummary` per goal (board always full); `retrieval_state` separates a grounded `miss` from degraded `not_retrieved` so a confident "not addressed" is never a false negative. With `batch=True` (the local-tier default via `quality_review.batch_goal_summaries`) all gate-passing goals are summarized in ONE LLM call (`_batch_goals`) instead of one-per-goal — retrieval + grounding stay per-goal, miss/not_retrieved need no LLM, and a goal the batch omits still renders (hit, no grounded summary); the board always shows every goal. On the full tier the per-goal calls run **concurrently** when `sub_concurrency > 1` (remote provider) — each goal keeps its own full-attention call (quality unchanged), board order preserved via indexed slots — and serially when 1 (local) |
| `_review_text.py` | `select_review_text(sections, full_text, budget, queries)` — budget-aware selection for the digest + rubric, replacing the naive `full_text[:budget]` slice that silently dropped the tail (Limitations/Conclusion). When the paper already fits the budget it returns the text **unchanged** (byte-identical to the old slice, so only over-cap papers change at all); otherwise it keeps referee-critical sections via **equal-share water-filling** (`_water_fill`: every high-value section gets ≥ `budget/n`, short critical sections taken whole, slack redistributed to long sections' heads — so an early 30 k Introduction can't starve the tail), skipping References/Acknowledgements/Appendix, then fills leftover budget with `PaperChunkIndex` BM25-ranked chunks. Errors propagate (no silent prefix fallback). `PaperChunkIndex` is imported lazily to keep this a leaf module (faithbench↔quality_review load-time cycle) |
| `_grounding.py` | the ONE quote-grounding module shared by `qa`, `_paper_goal_summaries`, and `quality_eval`, with two strictness levels by risk profile: **strict** (default — ≥6 words / ≥40 chars whitespace-normalized **verbatim** substring) for the safety-critical user-facing answers (`qa`, goal summaries; the bar faithbench validates — unchanged), and **`fuzzy=True`** (opt-in) for the quality CHECKLIST only — an NFKC-normalized token-`SequenceMatcher` that grounds a paraphrase whose tokens form contiguous runs covering ≥`FUZZY_MATCH_RATIO` (0.8) of the quote, while still rejecting hallucinated / scattered-word quotes.
