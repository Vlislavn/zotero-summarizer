# models — Pydantic contract (config + API shapes)

Every request/response body and the `goals.yaml` schema. Split by group; the
package `__init__` re-exports everything, so `from zotero_summarizer.models
import X` works unchanged.

```
config.py ──GoalsConfig──> api.py (AppState)
triage.py ──(domain)        all three ──*──> __init__.py  ──>  from ...models import X
```

| file | holds |
|---|---|
| `config.py` | `goals.yaml` schema: `GoalsConfig` + `LLMConfig`/`CorpusConfig`/`PrestigeConfig`/`ClassifierGateConfig`/… Migrates a legacy flat `llm:` block into `llm_routing` when the latter is absent. `CorpusConfig` carries the hybrid Library-search knobs (`bm25_enabled`/`reranker_enabled`/`reranker_model`, all local); `PrestigeConfig` carries the cold-start author-reputation prior (`cold_start_author_lift`/`cold_start_max_lift`/`cold_start_gamma`) — a capped, raise-only fallback to authors' field-normalized standing before a paper has its own citation percentile. `QualityReviewConfig.shadow_claim_check`/`claim_check_model` gate the Phase-A MiniCheck encoder shadow (off by default; needs the optional `minicheck` dep). `QualityReviewConfig` also carries the **tier-aware deep-review cost** knobs (`self_consistency_runs`/`lean_self_consistency_runs`/`lean_max_text_chars`/`batch_goal_summaries`) — a provider flagged `lean_deep_review` (e.g. ollama, prefill-bound) uses the cheaper `lean_*` caps + one batched goal-summary call, while every other provider (incl. MLX — loopback but fast) uses the full settings. The tier is keyed on the provider's `lean_deep_review` flag, NOT `is_local` (loopback ≠ lean). `QualityReviewConfig.prewarm_on_startup_k` (default 5, `0` disables; env override `ZS_DEEP_REVIEW_PREWARM_K`) drives the **launch-time deep-review prewarm** (`services/library/deep_review_prewarm.py`). `QualityReviewConfig.quality_band_primary` (default off; env override `ZS_QUALITY_BAND_PRIMARY`) flips the order-time quality lift from grade-only (the shipped default) to band-primary — a Phase-2-measured arm read by the shared `services/_common.band_primary_enabled` and applied via `model/rank_blend.quality_bonus`. `QualityReviewConfig.self_verification` (default on) gates the second-pass over-claim check on critical met items; `use_docling` (default off, optional `docling` dep) swaps the fitz PDF path for structured tables + figure captions. `review_web_articles` (default off) renders a web article (blog/Substack/news with HTML full text but no PDF) to a PDF so the fleet can review it (`_pdf_acquire`'s web-article rung; needs the `browser` extra). `max_pdf_bytes` defaults to **50 MB** (`QualityReviewConfig` + `FullTextRefineConfig`) — figure-heavy clinical/Nature PDFs run >20 MB. `RecoverAbstractConfig` (default ON: `goal_sim_threshold` 0.45, `max_per_tick` 3, `min_abstract_chars` 120) drives the daemon's **acquire-before-score rescue** — a gate-rejected, abstract-less, high-goal item (prestige-journal RSS ships no real abstract) gets its full text fetched + re-scored before the verdict stands (see `services/triage/README.md`). `UniversityAccessConfig` (off by default) configures the review fleet's **browser** PDF fetch for non-arXiv/paywalled papers: `enabled`, optional `ezproxy_prefix` (blank = SSO/OpenAthens, the persisted session carries access), `login_url`, `browser_profile_dir` (blank = `data/browser_profile` via Settings), `headless`, `fetch_timeout_secs`, `cookie_browser` (reuse an existing browser's session via `browser-cookie3` — chrome/firefox/edge/brave; Safari is unreadable on macOS 15+/26), and `browser_channel` (default `chrome`: drive the REAL Chrome binary so its fingerprint matches the injected `cf_clearance` and aggressive Cloudflare publishers like Nature/npj accept it; `""` = bundled chromium) — needs the optional `browser` extra (patchright + browser-cookie3). |
| `providers.py` | per-stage LLM routing: `ProviderConfig`/`ProviderType`, `StageModelConfig`, `LLMRoutingConfig`, `resolve_stage()` (stage → effective provider+model, inheriting `default`). `ProviderConfig.is_local` flags loopback endpoints (→ serial triage concurrency); `ProviderConfig.lean_deep_review` opts a provider into the cheaper deep-review tier (independent of `is_local` — set on ollama, left False on MLX); `ProviderConfig.max_sub_concurrency` caps how many deep-review SUB-calls (rubric samples / per-goal summaries) fan out in parallel on a remote provider (`None` → fall back to `TRIAGE_JOB_CONCURRENCY`; ignored for local providers, which stay serial). `ProviderConfig.temperature` (default `0.0`, the previously-hardcoded deterministic-triage value) is threaded into the openai-path `build_llm`; the native Anthropic adapter ignores it (Opus rejects the param). `ProviderConfig.thinking_effort` (`off`/`low`/`medium`/`high`, default `None` = leave untouched) is translated per backend dialect by `services/llm/thinking.py` (anthropic → thinking budget; plain OpenAI → `reasoning_effort`; vLLM/qwen `chat_template_kwargs` → `enable_thinking` on/off — graded collapses there). Pure data + lookups — no env reads, no client building. |
| `triage.py` | `SummarizeRequest/Response`, `TriageResult/Dimensions`, `QualityReview`, `PaperDigest` (now carries the full 11-part `summary_structure`: + executive_summary/key_findings/methods/limitations/industry+academy_impact), `GoalSummary` (per-goal board cell: `retrieval_state` hit/miss/not_retrieved + grounded summary/quotes/key_sections), `QualityEval` (reference-free 3-band quality verdict + decomposed rubric/evidence/red_flags/overstatements + `passes_agreed`/`passes_total` self-consistency count rendered as "N/M agree" + `claim_support_probs` = the Phase-A MiniCheck encoder shadow A/B record, empty unless `shadow_claim_check` on, never affects the band; + the type-aware coverage fields that REPLACE the unvalidated 1-5 self-scores: `paper_type`/`coverage_standard`/`coverage_met`/`coverage_applicable`/`coverage_fraction` [`grade` is DERIVED from this], `missing_critical`, and `self_verification_demoted` [critical items the 2nd pass overturned]), `ProposedVerdict` (the review-fleet's PRE-DECIDED reading verdict — `proposed` one of the 4 priorities + `confidence`/`rationale`/`flags` + `digest_read_decision`/`grade` provenance; a deterministic, LLM-free mapping of cached deep-review signals, surfaced on a reading-queue row as a SUGGESTION the human Confirms/Overrides — never auto-applied), batch + corpus + calibration models |
| `api.py` | HTTP request/response + write-path models (`*Request/*Response`, `AppState`, Zotero/pending shapes) |
| `setup.py` | the FROZEN `/api/setup/*` contract: `SetupStatusResponse` (+ nested `ConfigStatus`/`LlmStatus`/`PathStatus`/`PathEntry`/`ZoteroStatus`/`ClassifierStatus` + a `subsystems: list[SubsystemStatus]` runtime-readiness array, additive — a not-ready entry names the down subsystem + reason without affecting the top-level `ready`), `DetectedZoteroDir`/`DetectZoteroResponse`, `UpdatePathsRequest`/`UpdatePathsResponse`, `ValidateConfigRequest`/`ValidateConfigResponse` (+ `FieldError`/`ConnectionResult`). SECURITY: `api_key_env` is only ever an env-var NAME; key presence is the BOOL `api_key_present` — no model here carries a secret value |
| `__init__.py` | re-exports all of the above (the public `models` surface) |

**Boundaries:** depends only on `pydantic` + `domain`. Nothing here imports
`services/`/`storage/`/`api/`.
